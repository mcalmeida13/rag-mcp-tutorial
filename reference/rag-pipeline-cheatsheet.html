Definitions

Embedding

A numeric vector representing a piece of text, positioned so that semantically similar text has a similar vector. Produced here by all-MiniLM-L6-v2, dimension 384.

Chunk

A small, overlapping slice of a document (here: 50 words, 10-word overlap) — small enough to embed to a specific, searchable vector; large enough to stay coherent.

Vector index

Where chunk embeddings live alongside their source text and metadata, ready to be searched by similarity. Here: two parallel files, no database.

Cosine similarity

The standard way to compare two embedding vectors: the cosine of the angle between them. 1 = identical direction (very similar meaning), 0 = unrelated. Used in the retrieval step (next lesson).

Sliding-window chunk count formula

For a document of N words, chunk size C, overlap O, and step S = C - O:

num_chunks ≈ ceil((N - C) / S) + 1   (for N > C; else 1 chunk)

This is the formula behind the predict-then-reveal exercise in Lesson 1 — useful for sanity-checking chunk counts before running ingestion on a new corpus.
